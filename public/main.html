Brian's User Account Service

Why use this website

This website is designed to be a backend for a variety of other websites.

This website allows you to:

  • Create an account
  • Log into said account
  • Store data via that account
  • Retrieve data via that account

The advantages of using this site is:

  • One database to rule them all
  • Users are shared between your websites
  • All passwords are stored via a salted hash (using sha 512)
  • Comes with a simple metric tracking tool.

How to test this website

First create an account using the Create Account form. This will bring you to a webpage display the json response. Go back to this webpage to continue.
If you are a returning user go to the Log In form and log in.

When you are logged in you can click on the See Cookies link to see the cookie stored on your machine. You can also click on Remove Cookies to remove all cookies set by this website.

You can also change your password using the Change Password form. If you change your password then you must re-login on other computers, it is done automaticly on this computer.

Store some data form allows you to save some data linked to your user. This data will only be viewable by you when you are logged in. The path determines where the data is stored. Example: you could store 'Visitors' with a value of 100. Note: If you overwrite some data there is no way to get that data back.

Read some Data form reads the data that you request. If you used the above exapmle then reading 'Visitors' would return 100.

The Clear Logins link logs out all other computers using a cookie to log in.
